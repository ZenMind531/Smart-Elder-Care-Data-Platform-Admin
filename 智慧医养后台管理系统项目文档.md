# 智慧医养大数据公共服务平台云端后台管理系统项目文档

## 一、项目基本信息

**项目名称：** 智慧医养大数据公共服务平台云端后台管理系统

**英文名称：** Smart ElderCare Data Platform Admin

**GitHub 仓库名：** `smart-eldercare-data-platform-admin`

**项目类型：** Web 后台管理系统

**开发人数：** 4 人

**推荐技术栈：**

- 后端：Spring Boot、MyBatis-Plus、MySQL、Redis
- 前端：Vue / React、Element Plus / Ant Design
- 图表：ECharts
- 接口测试：Postman / Apifox
- 项目管理：Git、GitHub
- 部署环境：Windows / Linux / Docker

## 二、项目背景

随着老龄化程度加深，社区、养老机构和医疗服务机构需要对老年人的健康信息、健康预警、医生服务、设备状态和统计数据进行统一管理。传统人工管理方式效率低、数据分散、预警不及时，不利于及时发现重点人群和健康风险。

本项目拟开发一个智慧医养后台管理系统，通过信息化方式实现老人档案管理、健康预警管理、评估报告管理、重点人群管理、设备管理、医生账户管理、报表统计和数据看板等功能，提高医养服务管理效率。

## 三、项目目标

本系统主要面向医养服务机构后台管理员使用，目标是实现以下能力：

- 统一管理老人基础档案和健康信息。
- 对异常健康数据进行预警提示。
- 管理医生、老人账户和系统用户。
- 维护健康评估报告和重点人群信息。
- 管理智能设备基础信息和使用状态。
- 通过首页数据看板展示关键统计指标。
- 提供基础权限管理，保证不同用户访问不同功能。

## 四、用户角色

| 角色 | 说明 |
|---|---|
| 系统管理员 | 负责用户管理、角色权限、系统基础配置 |
| 医养机构管理员 | 负责老人档案、医生账户、设备、报表等业务管理 |
| 医生/工作人员 | 查看老人健康档案、处理健康预警、填写评估报告 |

## 五、核心功能模块

### 1. 登录与权限模块

- 用户登录、退出登录
- 登录状态校验
- 用户角色管理
- 菜单权限控制
- 修改密码

### 2. 首页数据看板

- 老人总数统计
- 医生数量统计
- 健康预警数量统计
- 设备数量统计
- 老人年龄分布图
- 健康风险等级分布图
- 近期预警列表

### 3. 老人档案管理

- 老人档案新增、删除、修改、查询
- 基础信息管理：姓名、性别、年龄、联系电话、家庭住址、紧急联系人
- 健康信息管理：血压、血糖、心率、既往病史、过敏史
- 档案详情查看

### 4. 健康预警管理

- 健康预警列表
- 根据血压、血糖、心率等数据生成预警
- 预警等级：低风险、中风险、高风险
- 预警状态：未处理、处理中、已处理
- 预警处理记录

### 5. 评估报告管理

- 新增健康评估报告
- 查看评估报告详情
- 按老人姓名、时间、风险等级查询报告
- 报告审核状态管理

### 6. 重点人群管理

- 重点老人列表
- 慢病老人、独居老人、高龄老人分类管理
- 重点关注原因记录
- 跟进状态管理

### 7. 设备管理

- 设备信息新增、删除、修改、查询
- 设备编号、设备名称、设备类型、绑定老人、设备状态
- 设备状态：正常、异常、停用
- 设备数据记录查看

### 8. 账户管理

- 老人账户管理
- 医生账户管理
- 管理员账户管理
- 账户启用、禁用
- 基础资料维护

### 9. 报表统计模块

- 老人数量统计
- 健康预警统计
- 医生服务统计
- 设备状态统计
- 按时间范围筛选统计数据
- 使用 ECharts 展示柱状图、折线图、饼图

## 九、项目展示重点

演示时建议重点展示以下流程：

1. 管理员登录系统。
2. 查看首页数据看板。
3. 新增一名老人档案。
4. 录入老人健康数据。
5. 系统根据异常数据生成健康预警。
6. 工作人员处理健康预警。
7. 生成或查看健康评估报告。
8. 查看报表统计图表。

## 十、项目优势

- 业务场景真实，符合智慧医养发展方向。
- 功能模块完整，适合四人分工协作。
- 技术难度适中，能体现 Java Web 项目开发能力。
- 首页数据看板和统计图表展示效果较好。
- 系统可以根据实训时间灵活控制功能范围。

## 十一、风险与解决方案

| 风险 | 解决方案 |
|---|---|
| 功能太多，时间不够 | 优先完成核心模块：登录、老人档案、健康预警、数据看板 |
| 大数据分析难度较高 | 使用 MySQL 聚合统计 + ECharts 图表展示，不做复杂大数据平台 |
| 前后端联调慢 | 提前约定接口字段，使用 Apifox / Postman 测试接口 |
| 组员代码冲突 | 使用 Git 分支开发，负责人统一合并 |
| 页面展示效果一般 | 首页重点做统计卡片、图表和近期预警列表 |
