# 实训项目推荐

## 推荐顺序

1. **7 智慧医养大数据公共服务平台云端后台管理系统**
2. **15 智能化在线教学支持服务平台学生中心系统**
3. **8 智慧医养大数据公共服务平台个人健康档案系统**
4. **11 健康大数据应用创新研发中心门户系统**
5. **6 互联网+智慧护理移动护理平台 App**

## 1. 项目 7：智慧医养大数据公共服务平台云端后台管理系统

**优点：**

- 功能完整，像真实后台管理系统，适合实训展示。
- 模块好分工：老人档案、健康预警、评估报告、设备管理、医生管理、报表统计都可以独立开发。
- 首页可以做数据看板，展示效果好，容易让老师觉得完成度高。
- 技术路线清楚：SpringBoot + MyBatisPlus + MySQL + Vue/前端框架即可完成。

**缺点：**

- 功能数量较多，需要提前控制范围。
- “大数据决策分析”不要做得太复杂，建议用 MySQL 统计 + ECharts 图表代替真正的大数据平台。

**结论：最推荐。完整、好分工、好展示。**

## 2. 项目 15：智能化在线教学支持服务平台学生中心系统

**优点：**

- 业务最熟悉，学生、课程、作业、考试、成绩这些功能很好理解。
- 不需要复杂行业背景，需求容易讲清楚。
- 适合做成完整流程：登录、选课、学习、作业、考试、成绩、笔记、问题反馈。
- 展示时容易演示，老师也容易看懂。

**缺点：**

- 功能较常见，需要页面做得完整一点，否则容易显得普通。
- 在线考试、成绩管理要设计好数据表，不然容易后期混乱。

**结论：第二推荐。最顺手，最不容易做偏。**

## 3. 项目 8：智慧医养大数据公共服务平台个人健康档案系统

**优点：**

- 范围比项目 7 小，更容易做完。
- 功能清晰：个人健康档案、健康报告、健康预警、医生专家库、健康咨询、积分、账户管理。
- 可以做成比较完整的个人端系统，页面展示效果不错。
- 健康预警可以用简单规则实现，例如血压、血糖、BMI 超过阈值就生成提醒。

**缺点：**

- 功能深度不如项目 7，后台管理感弱一些。
- 健康报告和预警如果只做静态页面，会显得比较空，需要配合数据录入和图表展示。

**结论：第三推荐。范围稳，适合保底做出高完成度。**

## 4. 项目 11：健康大数据应用创新研发中心门户系统

**优点：**

- 最稳的保底项目，主要是门户首页、新闻中心、通知公告、应用中心、政策、知识库、关于我们、内容管理。
- 技术难度相对低，适合快速完成。
- 页面数量够多，容易做出完整网站结构。
- 后台可以做内容管理，前台可以做门户展示，前后端都能体现工作量。

**缺点：**

- 创新性和业务深度一般，容易像普通新闻网站。
- 如果只做文章列表和详情页，展示效果会偏弱。
- 需要把首页、栏目页、后台管理做精致一点，避免显得简单。

**结论：第四推荐。最稳备选，适合在前几个都被抢时选择。**

## 5. 项目 6：互联网+智慧护理移动护理平台 App

**优点：**

- 功能很清楚：登录、注册、查看服务项目、预约下单、地址管理、订单、评价、投诉。
- 很像真实生活服务类 App，演示效果不错。
- 流程明确，可以做成“用户预约护理服务”的完整闭环。
- 如果会用 HBuilderX / UniApp，做移动端界面会比较加分。

**缺点：**

- 带 App / 小程序属性，如果组里没人熟悉移动端，会比普通 Web 后台麻烦。
- 订单流程要设计完整，否则容易只有页面没有业务逻辑。
- 难度标的是五星，虽然实际功能不算特别大，但移动端适配会增加工作量。

**结论：第五推荐。展示效果好，但最好在组里有人能做移动端时选择。**

## 最终建议

如果能抢，优先抢 **项目 7**。

如果项目 7 没了，选 **项目 15**。

如果项目 15 也没了，选 **项目 8**。

如果前三个都被抢，优先考虑 **项目 11**，因为它最稳。

如果组里有人会移动端或 UniApp，可以考虑 **项目 6**，展示效果会比项目 11 更好。
