## 二、业务背景与目标

### 背景
面向社区、养老机构和医疗服务机构，对老人的健康信息、健康预警、医生服务、设备状态和统计数据进行**统一信息化管理**，替代传统人工管理方式效率低、数据分散、预警不及时的问题。

### 目标
1. 统一管理老人基础档案和健康信息。
2. 对异常健康数据（血压/血糖/心率/血氧/体温等）自动预警。
3. 管理医生、老人、系统用户三类账户。
4. 维护健康评估报告和重点人群。
5. 管理智能设备基础信息和运行状态。
6. 通过首页数据看板展示关键统计指标。
7. 通过权限控制隔离不同角色可见功能。

### 角色

| 角色 | 说明 |
|---|---|
| 系统管理员 | 用户管理、角色权限、系统基础配置 |
| 医养机构管理员 | 老人档案、医生账户、设备、报表等业务管理 |
| 医生 / 工作人员 | 查看档案、处理预警、填写评估报告 |

## 六、API 封装规范

### 1. 通用约定（`src/api/http.ts`）

- 统一返回结构：
  ```ts
  interface ApiResponse<T> {
    code: number        // 0 / 200 视为成功
    message: string
    data: T
    timestamp?: string
  }
  ```
- 统一分页：
  ```ts
  interface PageResult<T> {
    records: T[]
    total: number
    page: number
    size: number
  }
  interface PageQuery {
    page?: number
    size?: number
    keyword?: string
  }
  ```
- `request<T>(url, options)`：自动注入 `Authorization`（从 `auth` store 取 token）、统一处理 `code !== 0/200` 抛错。
- `buildQuery(params)`：把对象拼成 query string。

### 2. 模块清单

| 文件 | 负责模块 | 主要接口（前端调用名） |
|---|---|---|
| `auth.ts` | 登录注册 | `login` / `register` / `logout` / `getProfile` |
| `roles.ts` | 角色 | `listRoles` / `createRole` / `updateRole` / `deleteRole` |
| `elderly.ts` | 老人档案 | 列表分页 / 详情 / 新增 / 修改 / 删除 / 导入导出 |
| `health.ts` | 健康数据 | 列表分页 / 新增记录 / 按老人查询历史 |
| `warnings.ts` | 健康预警 | 列表分页 / 处理 / 状态更新 / 趋势统计 |
| `devices.ts` | 设备 | 列表分页 / 绑定老人 / 状态更新 |

> 字段类型（如 `riskLevel: 'low' | 'medium' | 'high'`、`status: 'pending' | 'processing' | 'resolved'`、`deviceStatus: 'normal' | 'abnormal' | 'offline' | 'maintenance'`）已在对应 `api/*.ts` 中以 TS 类型定义，**后端在返 JSON 时要严格遵循这些值**。

```bash

# 启动开发服务器（默认 http://localhost:5173 ）
npm run dev

# 类型检查
npm run type-check

# 代码格式化
npm run format

# ESLint
npm run lint

# 生产构建（产物到 dist/）
npm run build

# 本地预览构建产物
npm run preview
```

### 3. 环境变量
参照 `.env.example` 在根目录建一个 `.env.local`（已在 `.gitignore`）：

```ini
# 后端 API 基础地址（按需改）
VITE_API_BASE_URL=http://localhost:8080/api
```

> ⚠️ 项目里 `vite.config.ts` 当前**没有**配置 `server.proxy`。
> 如果开发时遇到 CORS 报错，最快的解决办法是后端开启 CORS（推荐 `CorsFilter` 白名单）；
> 也可以在 `vite.config.ts` 加 `server.proxy` 把 `/api` 代理到后端。

## 附录 B：新增一个业务页面的标准流程

1. 在 `src/views/` 下新建 `XxxView.vue`。
2. 在 `src/router/index.ts` 的路由表里追加 `path/name/component`。
3. 在 `src/api/` 下新建 `xxx.ts`，用 `request` 封装接口、用 TS 定义返回类型。
4. 在 `src/stores/` 下新建 `useXxxStore.ts` 缓存数据（可选）。
5. 在 `src/components/dashboard/` 或 `src/components/` 下拆分子组件。
6. 在 `AppSidebar.vue` 的菜单里挂上跳转。
7. `npm run type-check` → `npm run dev` 自测。
